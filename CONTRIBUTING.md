# Contributing to Sales Insight Automator

First off, thank you for considering contributing to the **Sales Insight Automator**! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
- Check the [GitHub Issues](https://github.com/Anuj-er/Rabbit-Submission/issues) to see if the bug has already been reported.
- If it hasn't, open a new issue. Be as detailed as possible, including steps to reproduce the bug and your environment details.

### Suggesting Enhancements
- Open a new issue with a clear title and description of the suggested enhancement.
- Explain why this enhancement would be useful to most users.

### Pull Requests
- Fork the repository and create your branch from `main`.
- If you've added code that should be tested, add tests.
- Ensure the test suite passes.
- Make sure your code follows the existing style.
- Issue a Pull Request with a comprehensive description of the changes.

## Development Setup

See the [Setup & Installation Guide](documentation/setup.md) for detailed instructions on setting up your local environment.

### Backend (FastAPI)
- Use `flake8` for linting.
- Follow PEP 8 style guidelines.

### Frontend (Next.js)
- Use `npm run lint` before committing.
- Follow React best practices and use TailwindCSS for styling.

## Questions?
Feel free to open an issue or contact the maintainers.
